# GitHub初心者向け：浄徳寺サイトの公開手順

このファイルは、**GitHubの画面だけで公開する方法**を前提に書いています。  
難しい操作は使いません。

---

## 0. 先に結論

アップロードするのは、**ZIPファイルそのものではなく、解凍した中身**です。

アップロードするものは、次の一式です。

```text
index.html
404.html
styles.css
script.js
.nojekyll
favicon.svg
images フォルダ
```

---

## 1. ZIPをダウンロードして解凍する

まず、このサイト一式をダウンロードしてください。  
その後、PC上でZIPを解凍してください。

解凍後に、`index.html` や `images` フォルダが見えればOKです。

---

## 2. GitHubでリポジトリを開く

ブラウザで、あなたのリポジトリを開きます。

例：

```text
https://github.com/Yui9029/jotokuji-site
```

---

## 3. 既存ファイルを置き換える

### 方法
1. GitHubのリポジトリ画面で **Add file**
2. **Upload files** を押す
3. 解凍した中身をまとめてドラッグ＆ドロップ
4. 下へスクロールして **Commit changes** を押す

### 注意
- **ZIPファイルはアップロードしません**
- **解凍した中身をアップロードします**
- `images` フォルダも忘れずに入れてください
- 同じ名前のファイルがあれば、置き換えて大丈夫です

---

## 4. GitHub Pagesを有効化する

1. リポジトリ上部の **Settings** を開く
2. 左メニューまたはサイドバーから **Pages** を開く
3. **Build and deployment** のところで、以下を選ぶ

- **Source**：`Deploy from a branch`
- **Branch**：`main`
- **Folder**：`/ (root)`

4. **Save** を押す

---

## 5. 公開URLを確認する

通常は、数分待つと次のURLで公開されます。

```text
https://yui9029.github.io/jotokuji-site/
```

---

## 6. うまくいかないときの確認ポイント

### 画像が表示されない
- `images` フォルダがアップロードできていない
- `index.html` と `images` が同じ階層にない
- まだ反映待ち

### ページが更新されない
- `Commit changes` を押していない
- `Settings > Pages` が `main / root` になっていない
- ブラウザのキャッシュが残っている  
  → Windowsなら `Ctrl + F5` で再読み込み

### Pagesメニューが見つからない
- `Settings` の中にあります
- 画面幅が狭いと折りたたまれていることがあります

---

## 7. 今後よく使う編集場所

### 文章を直す
`index.html`

### デザインを直す
`styles.css`

### 写真を差し替える
`images` フォルダ

---

## 8. FBの出し方

確認するときは、次の観点で見てもらえると修正しやすいです。

- 第一印象で安心感があるか
- 電話したくなるか
- 文字が読みやすいか
- スマホで見ても崩れていないか
- 料金がわかりやすいか

---

## 9. 次におすすめの流れ

1. まずこのまま一式をアップロード
2. 実際の公開URLをスマホとPCで確認
3. 気になる点を箇条書きでFB
4. そのFBをもとに私がコードを再調整

この流れが、いちばん失敗が少ないです。
